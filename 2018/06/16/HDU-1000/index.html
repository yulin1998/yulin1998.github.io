发表于 2018-06-16 | 分类于 HDU

在学期的中旬，我突然决定要自己学习c语言，刚好我在csdn上发现了一位大佬，于是我问就问了问他，大佬对我说，直接去刷题，不需要看那些讲基础语法的书，如果做不出来就自己到网上去搜。我当时有点懵逼，心想大佬们难道都是这样的吗？不过想归想后来我就到了她给我推荐的网站去刷题了，大佬叫我从第11页开始写，但是我这个人有强迫症，所以我就从第一页开始写，因为我在这之前从来没有在这种online judge上刷过题，所以根本不知道里面的水有多深，不过当时我看第一题很简单，就是要求两个数相加求和，当时我心中大喜，然而就这一题我做了很久很久都没有AC（所谓的AC是指accepted，当你在hdu上面做完题的时候，需要在线提交，提交完以后，这个online judge系统会随机给出数据进行测试，如果对就会显示accepted，如果错误就会显示wrong answer），后来我不得已到网上去搜答案，发现不只是我，很多人都被这道题给坑害了，最主要的就是里面有一句关键语句不知道，导致虽然你的程序逻辑正确，但就是没办法AC。所以今天我就说一说那道题中的关键语句，那一道题如下：

Problem Description

Calculate A + B.

Input

Each line will contain two integers A and B. Process to end of file

Output

For each case, output A + B in one line.

Sample Input

1 1

Sample Output

2
