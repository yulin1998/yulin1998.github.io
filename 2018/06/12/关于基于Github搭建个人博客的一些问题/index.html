关于基于Github和Hexo搭建个人博客的方法

发表于 2018-06-12 | 分类于 正儿八经

经过好几天的探索，我终于搭建了一个个人博客，但是这个个人博客是没有域名的，它的使用是基于GitHub pages的，至于域名的话，我想等我哪天暴富了，可以去买一个，到时候就要方便的多，但是我其实觉得现在这个样子也挺好的，至少很拉风嘛。

都说程序猿需要一个自己的博客，一个是为自己的学习路提供帮助，一个是用来装载一些是人都会有的瞎扯淡，对于我来说毫无疑问，肯定是后者要多一些，至于前者的话应该不会很多，关于一些技术的问题我更多的可能会放在csdn的博客上，因为上面大佬比较多，写错了有人指正，不管怎么说，在这个博客搭建完成的时候，我总算是感觉有了自己的一方天地，这种感觉挺爽的。本来还想给这个博客搞一个站点浏览的统计以及一个评论区域，但后来想一想，这样挺费时间的，估计我又得弄好几天，但最主要的是，我这个个人博客更多的还是写给自己看，让自己养成一个写东西的好习惯，所以站点统计和评论区域对我来说就显得不是很重要了，包括这个主题我都是选择的最简洁的，但是从技术角度来看的话，别看它页面看起来简洁，要自己写的话比那些图片很多，看起来特别高大上的网页难多了，不过现在我倒不用操心这种问题，因为我都还写不出来。在说这个过程之前，我要说一说自己是怎么心血来潮突然想要搭建一个个人博客的。这个念头其实早就有了，但是当时我一直都认为这个很麻烦，会很花费时间，所以一推再推，但我在学习GitHub以及git的操作时，突然想到或许自己可以试一试，而且那天刚好也只有一节课，很显然我太天真了，就因为这个想法那一天从早上第一节课上完一直到晚上的十一点，我一直在弄这个东西，最气人的是我还没有弄好，浏览器里开了十几个页面，全都是查相关的操作，以前在赏析莎士比亚的作品的时候总会说，一千个读者就有一千个哈姆雷特，我觉的这句话改一改放到电脑上也挺适用，一千台电脑就有一千个问题，反正这个过程中总结起来就是按照一片文章执行操作，然后出现了这篇文章所没有提到的问题，然后又去找另外一篇文章来解决这个问题，然后又重复上述，以至于在这个过程中我无数次的想砸电脑，但是仔细一想电脑是无辜的，只是自己太蠢了，一想到这又更气了，不过最终还好，总算是解决了那些问题，搭建了一个极其简洁，很符合我的风格的个人博客，所以接下来我就说一说搭建的整个过程吧。

我是基于macos系统搭建的，所以这一些经验的话应该对使用macOS系统的人最受用，但是这不代表使用windows的没办法，如果你去网上找的话，你会发现基于windows要比基于macOS的多得多，好了废话我就不说了，我就直接开始吧（未完待续。。。今天就写到这了，剩下的下次写，可能下次也不一定写得完，不过一旦写完肯定是很详细的）
